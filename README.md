# [@NaokiKosuge](https://github.com/NaokiKosuge) のポートフォリオサイト

このプロジェクトは、@NaokiKosuge のポートフォリオサイトです。[Docusaurus 2](https://docusaurus.io/) を使用して作成されています。

## インストール

```console
npm run install
```

## ローカル開発

```console
npm run start
```

This command starts a local development server and opens up a browser window. Most changes are reflected live without having to restart the server.

## ビルド

```console
npm run build
```

This command generates static content into the `build` directory and can be served using any static contents hosting service.

## デプロイ

```console
GIT_USER=<Your GitHub username> USE_SSH=true npm run deploy
```

If you are using GitHub pages for hosting, this command is a convenient way to build the website and push to the `gh-pages` branch.

## ドキュメントの書き方

### ドキュメントファイルの作成

ドキュメントは[`docs/`](docs/) ディレクトリ内に任意のディレクトリ階層の後、`[name].md`を作成します。

### ドキュメントのメタデータ

ドキュメントごとに次のメタデータフィールドを使用できます。

- `id`: 一意のドキュメントID。このフィールドが存在しない場合、ドキュメントの`id`はデフォルトでファイル名（拡張子なし）になります。
- `title`: ドキュメントのタイトル。このフィールドが存在しない場合、ドキュメントの`title`はデフォルトでその`id`になります。
- `hide_title`: ドキュメントの上部にあるタイトルを非表示にするかどうか。デフォルトは`false`です。
- `hide_table_of_contents`: 右側の目次を非表示にするかどうか。デフォルトでは`false`です。
- `sidebar_label`: ドキュメントのサイドバーと、このドキュメントの[次へ/前へ]ボタンに表示されるテキスト。このフィールドが存在しない場合、ドキュメントの`sidebar_label`はデフォルトでその`title`になります。
- `sidebar_position`: 自動生成されたサイドバーアイテムを使用するときに、生成されたサイドバースライス内のドキュメントの位置を制御することを許可します。 IntまたはFloatにすることができます。
- `parse_number_prefixes`: ドキュメントに番号プレフィックス（`001 - My Doc.md`、`2. MyDoc.md`...）がある場合、プラグイン`numberPrefixParser`によって自動的に解析および抽出され、番号プレフィックスが`sidebar_position`として使用されます。このドキュメントで番号プレフィックスの解析を無効にするには、`parse_number_prefixes: false`を使用します。
- `custom_edit_url`: このドキュメントを編集するためのURL。このフィールドが存在しない場合、ドキュメントの編集URLは、`docusaurus-plugin-content-docs`に渡されたオプションフィールドから`editUrl`にフォールバックします。
- `keywords`: 検索エンジン用のドキュメントページのキーワードメタタグ
- `description`: ドキュメントの説明。`<head>`の`<meta name="description" content="..." />`および`<meta property="og:description" content="..." />`になります。検索エンジンで使用されます。このフィールドが存在しない場合、デフォルトでコンテンツの最初の行になります。
- `image`: 投稿へのリンクを表示するときに使用されるカバーまたはサムネイル画像
- `slug`: スラッグ。ドキュメントのURL（`/<routeBasePath>/<slug>`）をカスタマイズできます。

例:

```markdown
---
id: doc-markdown title: Markdown Features hide_title: false hide_table_of_contents: false sidebar_label: Markdown :)
custom_edit_url: https://github.com/facebook/docusaurus/edit/master/docs/api-doc-markdown.md
description: How do I find you when I cannot solve this problem keywords:

- docs
- docusaurus image: https://i.imgur.com/mErPwqL.png
  slug: /myDoc

---
My Document Markdown content
```

記述形式:

```
:::[種類] [タイトル]

本文

:::
```

種類:

- `note`: ℹ️NOTE
- `tip`: 💡TIP
- `info`: ℹINFO
- `caution`: ⚠️CAUTION
- `danger`: 🔥DANGER

## サイドバーの作成方法

サイドバーの設定は[`sidebar.js`](sidebars.js) に記述します（`docusaurus.config.js`で指定しているため）。

```js title="sidebar.js"
module.exports = {
  mySidebar1: [
    /* サイドバーアイテム */
  ],
  mySidebar2: [
    /* サイドバーアイテム */
  ],
  // ...
};
```

### サイドバーアイテム

- Doc: ドキュメントへのリンク
- Ref: ドキュメントへのリンク（但しドキュメントページにはそのドキュメントはサイドバーに割り当てられない）
- Link: 外部または内部のリンク
- Category: 階層を作成する
- Autogenerated: ファイルシステムベースで自動生成

```js title="sidebar.js"
module.exports = {
  sidebar: [
    // Doc (通常のシンタックス)
    {
      type: 'doc',
      id: 'doc1'
    },
    // Doc (ショートハンドシンタックス)
    'doc2',  // document id
    
    // Ref
    {
      type: 'ref',
      id: 'doc1',
    },
    
    // Link (外部リンク)
    {
      type: 'link',
      label: 'Facebook',
      href: 'https://facebook.com',
    },
    // Link (内部リンク)
    {
      type: 'link',
      label: 'Home',
      href: '/',
    },
    
    // Category (通常のシンタックス)
    {
      type: 'category',
      label: 'Guides',
      collapsed: false,
      items: [
        'creating-pages',
        {
          type: 'category',
          label: 'Docs',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
    // Category (ショートハンドシンタックス)
    'creating-pages',
    {
      Docs: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
    },
      
      // Autogenerated
    {
      type: 'autogenerated',
      dirName: '.',  // 「.」は現在のドキュメントフォルダを意味します。
    },
  ],
};
```

### サイドバーの非表示

```js title="docusaurus.config.js"
module.exports = {
  themeConfig: {
    hideableSidebar: true,
  },
};
```

## ブログの書き方

### ブログ記事ファイルの作成

ブログ記事は[`blog/`](blog/) ディレクトリ直下に`YYYY-MM-DD-[slug].md`という形式の名前で作成します。

たとえば、`my-website/blog/2019-09-05-hello-docusaurus-v2.md`：

```markdown
---
title: Welcome Docusaurus v2
author: Joel Marcey
author_title: Co-creator of Docusaurus 1
author_url: https://github.com/JoelMarcey
author_image_url: https://graph.facebook.com/611217057/picture/?height=200&width=200
tags: [hello, docusaurus-v2]
description: This is my first post on Docusaurus 2.
image: https://i.imgur.com/mErPwqL.png
hide_table_of_contents: false
---
Welcome to this blog. This blog is created with [**Docusaurus 2 alpha**](https://docusaurus.io/).

<!--truncate-->

This is my first post on Docusaurus 2.

A whole bunch of exploration to follow.
```

### ブログ記事のメタデータ

必須フィールドは`title`のみです。ただし、他のオプションとともに、ブログ記事に作成者情報を追加するオプションも提供しています。

- `author`: 表示される作成者名
- `author_url`: 著者名がリンクされるURL。これは、GitHub、Twitter、FacebookのプロファイルURLなどである可能性があります。
- `author_image_url`: 著者のサムネイル画像へのURL。
- `author_title`: 著者の説明
- `title`: ブログ記事のタイトル
- `tags`: ブログ記事にタグ付けする文字列のリスト
- `draft`: ブログ記事が進行中であり、したがってまだ公開されるべきではないことを示すbooleanフラグ。ただし、ドラフトブログ投稿は開発中に表示されます。
- `description`: 投稿の説明。これは、`<head>`の`<meta name="description" content="..." />`および`<metaproperty = "og：description" content="..." />`になります。検索エンジンで使用されます。このフィールドが存在しない場合、デフォルトでコンテンツの最初の行になります。
- `image`: 投稿へのリンクを表示するときに使用される表紙またはサムネイル画像
- `hide_table_of_contents`: 右側の目次を非表示にするかどうか。デフォルトでは`false`です。

### 要約の切り捨て

ブログ投稿で`<！-truncate->`マーカーを使用して、公開されているすべてのブログ投稿を表示するときに要約として表示される内容を表します。 `<！-truncate->`より上のものはすべて、要約の一部になります。例えば:

```markdown
---
title: Truncation Example
---
All these will be part of the blog post summary.

Even this.

<!--truncate-->

But anything from here on down will not be.

Not this.

Or this.
```

TypeScript

TypeScript は、型の構文を備えた JavaScript です。

Node.js 環境における TypeScript プロジェクト​

  1. パッケージのインストール
    # TypeScript コア
    npm i -D typescript
    # 型定義ファイル
    npm i -D @types/...
  2. 構成ファイル (tsconfig.json) の作成
  3. マニフェスト (package.json) の変更
  4. コンパイル

型エイリアスとインターフェースの違い

型エイリアスとインターフェースは非常によく似ており、多くの場合、自由に選択することができます。インターフェイスのほぼすべての機能が型でも利用できますが、重要な違いは、常に拡張可能なインターフェイスに対して、型は新しいプロパティを追加するために再オープンすることができないという点です。

【比較】拡張の違い
インターフェースの拡張
interface Animal {
  name: string
}

interface Bear extends Animal {
  honey: boolean
}

const bear: Bear = {
  name: 'Ted',
  honey: true,
}
交差による型の拡張
type Animal = {
  name: string
}

type Bear = Animal & {
  honey: boolean
}

const bear: Bear = {
  name: 'Ted',
  honey: true,
}
【比較】プロパティの追加の違い
インターフェース宣言のマージ
interface User {
  name: string
}

interface User {
  age:  number
}

const user: User = {
  name: 'John',
  age: 30,
}
エラー：作成後にタイプを変更することはできません
type user = {
  name: string
}

type User = {
  age: number
}

// Error: Duplicate identifier 'User'.

ユーティリティ型​

TypeScript は、一般的な型変換を容易にするためのいくつかのユーティリティ型を提供します。これらのユーティリティはグローバルに利用できます：

  • Partial<T> – T のすべてのプロパティをオプションに設定した型を構築します。
  • Required<T> – T の持つ全てのプロパティを持つ型を構築します。
  • Readonly<T> – T のすべてのプロパティが読み取り専用に設定されたタイプを構築します。
  • Record<K, T> – キーが K で、値が T であるオブジェクトタイプを構築します。
  • Pick<T, K> – T からキー K のセットを選択した型を構築します。
  • Omit<T, K> – T からキー K を取り除いた型を構築します。
  • Exclude<U, E> – Union 型 U から Union 型 E を取り除いた型を構築します。
  • Extract<T, U> – Union 型 T から Union 型　U を取り出した型を構築します。
  • NonNullable<T> – T から null 及び undefined を取り除いた型を構築します。
  • Parameters<F> – 関数型 F のパラメーターで使用される型からタプル型を構築します。
  • ConstructorParameters<C> – コンストラクター関数型 C のパラメーターで使用される型からタプル型を構築します。
  • ReturnType<F> – 関数型 F の戻り型で構成される型を構築します。
  • InstanceType<C> – コンストラクタ型 C のインスタンス型で構成される型を構築します。
  • ThisParameterType<F> – 関数型 F の this パラメーター型を抽出します。
  • OmitThisParameter<F> – 関数型 F の this パラメーター型を取り除いた型を構築します。
  • ThisType<T>
  • Uppercase<StringType> – 文字列型 Stringtype を大文字にした型構築します。
  • Lowercase<StringType> – 文字列型 Stringtype を小文字にした型構築します。
  • Capitalize<StringType>
  • Uncapitalize<StringType>

TSConfig​

🔗 TypeScript: TSConfig Reference - Docs on every TSConfig option

ディレクトリ内の TSConfig ファイルは、そのディレクトリが TypeScript または JavaScript プロジェクトのルートであることを示しています。 TSConfig ファイルは tsconfig.json または jsconfig.json のいずれかであり、どちらも同じ構成変数のセットを持っています。

  • files

    プログラムに含めるファイルの許可リスト

  • extends

    継承元の別の構成ファイルへのパスを含む文字列

  • include

    プログラムに含めるファイル名またはパターンの配列を指定します。

  • exclude

    include を解決するときにスキップする必要があるファイル名またはパターンの配列を指定します。

  • references

    TypeScriptプログラムをより小さな部分に構造化する方法
